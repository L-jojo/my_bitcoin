Performance

Charts below present system memory and time needed to sync BTC block chain up to the given block number.

Note:
Gocoin uses (configurable) LastTrustedBlock to avoid verifying scripts from all blocks known as valid.
Referenced Bitcoin Core uses an identical mechanism (configurable with assumevalid parameter).
Both implementations come with a recent trusted block built into their code (no need to configure them explicitly).

Gocoin 1.10.1, built with Go 1.18.3

Hetzner dedicated server with 3.4GHz Intel i7-6700 CPU, 64GB RAM and 2x2TB HDD

With comparision to Bitcoin Core 23.0:

Both the clients were using their default configuration.

Hetzner dedicated server with 3.6GHz Intel i7-7700 CPU, 64GB RAM and 2x512GB SSD

Default config:


Time needed to re-build UTXO database from blocks stored on the disk: 1.2 hours (up to block #700000)

Tweaked config:


Time needed to re-build UTXO database from blocks stored on the disk: 55 minutes (up to block #700000)
Changed config values:
  • Net.ListenTCP: false (disallow incomming connections as they are often bad actors - just to be safe)
  • Memory.GCPercTrshold: 300 (to use up to 64GB of RAM)
  • Memory.UseGoHeap: true (works together with the above setting)
  • Memory.CacheOnDisk: false (further increases RAM usage, but makes the biggest difference)
  • Stat.NoCounters: true (does not seem to make any difference)
  • IP addresses of nodes to connect were manually preselected and provided through the friends.txt file
